## 一、前置準備

### 您需要準備：

- [ ] **Google 帳號**（用於 Firebase、Cloud Run）
- [ ] **LINE 帳號**（用於測試）
- [ ] **電腦已安裝**：
  - [Node.js 20 以上](https://nodejs.org/)（建議 LTS 版）
  - [Git](https://git-scm.com/)（若要用版本控制）

### 檢查 Node.js 是否安裝成功

打開「命令提示字元」或「PowerShell」，輸入：

```
node -v
```

若顯示 `v20.x.x` 或更高版本，代表安裝成功。

## 二、建立 Firebase 專案

### 步驟 2-1：進入 Firebase Console

1. 開啟瀏覽器，前往：**https://console.firebase.google.com/**
2. 使用您的 Google 帳號登入

### 步驟 2-2：建立新專案

1. 點擊「**新增專案**」（或「建立專案」）
2. **專案名稱**：輸入 `golf-coach`（或您喜歡的名稱）
3. 是否啟用 **Google Analytics**：可選「否」以簡化設定
4. 點擊「**建立專案**」
5. 建立完成後，點「**繼續**」進入專案總覽

### 步驟 2-3：啟用 Firestore 資料庫

1. 左側選單點「**建置**」→「**Firestore Database**」
2. 點擊「**建立資料庫**」
3. 選擇模式：**正式環境**（Production）
4. 選擇位置：**asia-east1**（台灣）
5. 點擊「**啟用**」

### 步驟 2-4：設定 Firestore 安全規則

1. 在 Firestore 頁面，點上方「**規則**」分頁
2. 將規則改為以下內容（複製貼上）：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

3. 點擊「**發佈**」

> 說明：此規則代表「不允許前端直接存取」，所有資料透過後端 API 操作，較安全。

## 三、取得 Firebase 金鑰

### 步驟 3-1：產生服務帳號金鑰

1. 在 Firebase Console 左側，點擊「**專案設定**」（齒輪圖示）
2. 點選「**服務帳戶**」分頁
3. 往下捲動，點擊「**產生新的私密金鑰**」
4. 彈出視窗點「**產生金鑰**」
5. 瀏覽器會下載一個 JSON 檔案

### 步驟 3-2：放置金鑰檔案

1. 將下載的 JSON 檔案**重新命名**為：`serviceAccountKey.json`
2. 將檔案**移動**到專案資料夾：`d:\Golf-Coach\rebuild\`
3. 確認路徑為：`d:\Golf-Coach\rebuild\serviceAccountKey.json`

### 步驟 3-3：確認 .gitignore（重要！）

1. 開啟 `d:\Golf-Coach\rebuild\.gitignore`
2. 確認裡面有這一行：`serviceAccountKey.json`
3. 若沒有，請手動加入

> ⚠️ 此金鑰檔案**絕對不能**上傳到 GitHub 或任何公開地方，否則他人可存取您的資料庫。

## 四、建立 LINE 頻道與 LIFF

### 步驟 4-1：進入 LINE Developers Console

1. 前往：**https://developers.line.biz/console/**
2. 使用 LINE 帳號登入
3. 若為首次使用，需先建立「Provider」（提供者），名稱可填 `golf-coach`

### 步驟 4-2：建立 Messaging API 頻道

1. 在 Provider 下，點擊「**建立**」→ 選擇「**Messaging API**」
2. 填寫：
   - **頻道名稱**：`高爾夫預約`（或自訂）
   - **頻道說明**：可留空
   - **類別**：選「教育」
   - **子類別**：隨意
3. 勾選同意條款，點「**建立**」
4. 建立完成後，進入該頻道

### 步驟 4-3：取得 Channel Access Token 與 Channel Secret

1. 在頻道頁面，點選「**Messaging API**」分頁
2. 往下找到「**Channel access token**」：
   - 若顯示「Issue」，點擊後會產生一組 Token
   - **複製**這組 Token，稍後會用到
3. 點選「**Basic settings**」分頁：
   - 找到「**Channel secret**」
   - **複製**這組 Secret，稍後會用到

### 步驟 4-4：設定 Webhook URL（先跳過，部署後再填）

- 在 Messaging API 分頁，找到「Webhook URL」
- **先留空**，等後端部署完成後再回來填寫
- 將「**Use webhook**」設為「**Enabled**」

### 步驟 4-5：建立 LIFF 應用程式

1. 在頻道頁面，點選「**LIFF**」分頁
2. 點擊「**新增**」
3. 填寫：
   - **LIFF app name**：`教練後台`
   - **Size**：選擇「Full」
   - **Endpoint URL**：先填 `https://example.com`（之後部署時再改）
   - **Scope**：勾選「profile」
4. 點擊「**新增**」
5. 新增完成後，會顯示「**LIFF ID**」（例如：`1234567890-abcdefgh`）
6. **複製**這組 LIFF ID，稍後會用到

### 步驟 4-6：取得教練的 LINE User ID

1. 將您的 LINE 帳號**加入**剛建立的頻道為好友（掃描頻道 QR Code）
2. 在 Messaging API 分頁，找到「**Webhook URL**」下方的「**Webhook settings**」
3. 點擊「**Verify**」旁的「**Edit**」，可看到測試用 Webhook
4. 或：先完成後端部署，當您傳訊息給 Bot 時，後端 log 會印出您的 `userId`
5. **暫時**：可先不填，等測試時從 log 取得

## 五、設定環境變數

### 步驟 5-1：建立 .env 檔案

1. 在 `d:\Golf-Coach\rebuild\` 資料夾中，找到 `.env.example` 檔案
2. **複製**此檔案，並重新命名為 `.env`
3. 用文字編輯器（如 VS Code、記事本）開啟 `.env`

### 步驟 5-2：填入各項數值

將 `.env` 內容改為（請替換成您自己的數值）：

```env
# LINE（從步驟 4-3 取得）
LINE_CHANNEL_ACCESS_TOKEN=您的Channel_Access_Token
LINE_CHANNEL_SECRET=您的Channel_Secret

# LIFF（從步驟 4-5 取得）
LIFF_COACH_ID=您的LIFF_ID

# Firebase（金鑰檔案路徑）
GOOGLE_APPLICATION_CREDENTIALS=./serviceAccountKey.json

# Cron 密鑰（可自訂一組亂碼，例如：abc123xyz）
CRON_SECRET=請輸入一組自訂密鑰

# 教練白名單（您的 LINE userId，逗號分隔，多個教練用逗號隔開）
COACH_LINE_USER_IDS=您的LINE_User_ID
```

### 步驟 5-3：確認 .env 不被上傳

1. 開啟 `d:\Golf-Coach\rebuild\.gitignore`
2. 確認有 ` .env` 這一行
3. 若沒有，請加入

## 六、本機測試

### 步驟 6-1：安裝依賴

1. 開啟「命令提示字元」或「PowerShell」
2. 輸入以下指令切換到專案目錄：

```
cd d:\Golf-Coach\rebuild
```

3. 安裝套件：

```
npm install
```

4. 等待安裝完成（約 1–2 分鐘）

### 步驟 6-2：建置專案

```
npm run build
```

若看到沒有錯誤訊息，代表建置成功。

### 步驟 6-3：啟動後端

```
npm start
```

若成功，會顯示類似：

```
Server listening at http://0.0.0.0:8080
```

### 步驟 6-4：測試 Webhook（需 ngrok 或類似工具）

**方式 A：使用 ngrok（推薦）**

1. 下載 [ngrok](https://ngrok.com/)
2. 執行：`ngrok http 8080`
3. 會得到一個網址，例如：`https://abc123.ngrok.io`
4. 到 LINE Developers Console → Messaging API → Webhook URL
5. 填入：`https://abc123.ngrok.io/webhook`
6. 用手機 LINE 傳訊息給您的 Bot，後端 log 會顯示收到的訊息
7. 同時可從 log 取得您的 `userId`，複製到 `.env` 的 `COACH_LINE_USER_IDS`

**方式 B：先不測試 Webhook**

- 可先跳過，直接測試 LIFF 頁面（需先設定 LIFF 的 Endpoint URL 為可存取的網址）

### 步驟 6-5：設定教練 LIFF 頁面

1. 用文字編輯器開啟：`d:\Golf-Coach\rebuild\liff\coach\index.html`
2. 在檔案開頭（約第 9 行附近）找到設定區塊：

```html
window.LIFF_ID = '';           // 例：'1234567890-abcdefgh'
window.API_URL = 'http://localhost:8080';
```

3. 將 `window.LIFF_ID` 改為您從步驟 4-5 取得的 LIFF ID（例如：`'1234567890-abcdefgh'`）
4. 本機測試時，`window.API_URL` 保持 `'http://localhost:8080'` 即可
5. 若使用 ngrok，可改為：`window.API_URL = 'https://您的ngrok網址';`

### 步驟 6-6：在本地預覽 LIFF 頁面

1. LIFF 必須在 **HTTPS** 環境下執行
2. 可用 VS Code 的「Live Server」擴充功能，或任何可提供 HTTPS 的本地伺服器
3. 或：先部署到 Cloud Run 後，再透過 LINE 開啟 LIFF

## 七、部署到 Cloud Run（選做）

當本機測試沒問題後，可將後端部署到 Google Cloud Run，讓 LINE Webhook 與 LIFF 能從網際網路存取。

### 前置：安裝 Google Cloud CLI

1. 下載並安裝 [Google Cloud SDK](https://cloud.google.com/sdk/docs/install)
2. 安裝完成後，開啟終端機執行：`gcloud init`
3. 登入 Google 帳號，選擇專案（需與 Firebase 同一個專案）

### 步驟 7-1：建立 Cloud Run 環境變數檔案

1. 在 `d:\Golf-Coach\rebuild` 目錄，複製 `env.production.yaml.example` 為 `env.production.yaml`
2. 用文字編輯器開啟 `env.production.yaml`，填入與 `.env` 相同的數值（**勿提交到 Git**）
3. 若您已有 `.env`，可參考其內容填入；格式為 `KEY: "值"`（值需用雙引號包住）

### 步驟 7-2：授予 Cloud Run 存取 Firestore 的權限

Cloud Run 使用專案預設服務帳號，需授予 Firestore 存取權限。在終端機執行：

```
gcloud projects add-iam-policy-binding golf-coach-aebcd --member="serviceAccount:637405580782-compute@developer.gserviceaccount.com" --role="roles/datastore.user"
```

> 若您的專案 ID 或專案編號不同，請替換為正確數值。專案編號可在 [Google Cloud Console](https://console.cloud.google.com/) 專案選單中查看。

### 步驟 7-3：執行部署

在 `d:\Golf-Coach\rebuild` 目錄執行：

```
gcloud run deploy golf-coach-api --source . --region asia-east1 --allow-unauthenticated --env-vars-file env.production.yaml
```

### 步驟 7-4：部署完成後

1. 會得到一個網址，例如：`https://golf-coach-api-xxxxx-asia-east1.run.app`
2. 將此網址記下，用於：
   - **LINE Webhook URL**：`https://您的網址/webhook`
   - **LIFF 頁面**：將 `liff/coach/index.html` 中的 `window.API_URL` 改為此網址
3. LIFF 靜態頁面需另外部署（可放 Firebase Hosting），並在 LINE Developers Console 的 LIFF 設定中填入該頁面的 HTTPS 網址

### 若部署失敗：「Container import failed」

通常是**環境變數未正確傳入**。請確認：
- `env.production.yaml` 已建立且格式正確
- 部署指令中有加上 `--env-vars-file env.production.yaml`
- 或到 [Cloud Run Console](https://console.cloud.google.com/run) → 您的服務 →「編輯與部署新修訂」→「變數與密碼」→ 手動加入所有環境變數

## 八、設定定時任務（選做）

用於「預約提醒」「到期提醒」「教練行程彙整」等自動推播。

1. 前往 [Cloud Scheduler](https://console.cloud.google.com/cloudscheduler)
2. 建立三個工作，設定如下：

| 工作名稱 | URL | 排程 | 時區 |
|----------|-----|------|------|
| reminders | `https://您的CloudRun網址/api/cron/reminders` | `0 21 * * *` | Asia/Taipei |
| expiry | `https://您的CloudRun網址/api/cron/expiry` | `0 8 * * *` | Asia/Taipei |
| coach-digest | `https://您的CloudRun網址/api/cron/coach-digest` | `0 21 * * *` | Asia/Taipei |

3. 每個工作的 HTTP 方法：**POST**
4. 在「進階」中新增 Header：`X-Cron-Secret` = 您的 `CRON_SECRET` 值

## 常見問題

### Q1：npm install 失敗怎麼辦？

- 確認 Node.js 版本：`node -v` 需為 20 以上
- 嘗試刪除 `node_modules` 資料夾後，重新執行 `npm install`

### Q2：找不到 serviceAccountKey.json？

- 確認檔案放在 `d:\Golf-Coach\rebuild\` 底下
- 檔名必須是 `serviceAccountKey.json`（區分大小寫）

### Q3：LIFF 頁面顯示「Invalid LIFF ID」？

- 確認 `window.LIFF_ID` 是否正確
- LIFF 必須在 HTTPS 環境下執行

### Q4：傳訊息給 Bot 沒反應？

- 確認 Webhook URL 已正確設定
- 確認 ngrok 或 Cloud Run 服務正在運行
- 查看後端 log 是否有錯誤訊息
