# 高爾夫教練系統 - 專屬影片分享功能規劃

> **建立時間**：2026-03-21
> **功能目的**：讓教練有一個空間能集中管理專業教學影片，並可針對「特定學員」分享指定的影片內容。

---

## 1. 影片儲存與播放方案
- **選定方案**：**YouTube（非公開影片）**
- **運作方式**：教練將影片檔案上傳至自己的 YouTube 頻道，將隱私設定為「非公開」（只有擁有連結的人能觀看）。接著，教練只要在系統的後台將該影片的「YouTube 連結」貼上即可。
- **優點評估**：
  - 完全沒有流量與儲存空間的費用。
  - YouTube 提供極佳的串流播放品質（自動因應網速調整畫質）。
  - 對教練來說操作門檻低。

## 2. 資料庫與權限設計 (Firebase Firestore)
為了管理哪些學員可以看哪些影片，系統需要建立以下資料結構：
- **`videos` (影片總庫)**：
  - 用來儲存影片的基本資訊，例如：`videoId`, `title` (影片標題), `youtubeUrl` (完整網址或影片ID), `description` (文字說明), `createdAt` (建立時間)。
- **權限對應設計 (`video_access`)**：
  - 透過獨立的集合 (collection) 或是掛在學員資料下，來記錄特定 `userId` (學員) 擁有特定 `videoId` (影片) 的觀看權限。

## 3. 教練端介面與功能 (Coach LIFF)
- **影片庫管理頁面**：
  - 提供教練介面，用來新增影片（輸入標題、YouTube 連結、註解），並能瀏覽或刪除已加入的影片。
- **影片專屬分享功能**：
  - 每支影片旁邊會配備一個「分享 / 派發」按鈕。
  - 點擊後，會彈出目前所有學員的清單。教練可以勾選（可多選）特定學員，按下確認後，系統便會建立對應的觀看權限。

## 4. 學員端介面與通知 (Student LIFF & LINE Bot)
- **即時推播通知 (LINE Bot)**：
  - 當教練分享影片後，系統會自動觸發 LINE Bot，傳送個人化訊息給被選中的學員，告知：「您的教練剛才為您準備了專屬教學影片『[影片標題]』，點擊下方選單立即觀看！」。
- **我的專屬影片區 (Student LIFF)**：
  - 學員點開 LIFF 介面後，會新增一個「專屬影片」的分頁。
  - 這個頁面將即時撈取該學員被授權的影片清單，學員點擊後可直接在網頁內透過 YouTube 內嵌播放器觀看影片學習。
